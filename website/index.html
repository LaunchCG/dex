dex.

The package manager for AI coding agents.

One config file. Every AI platform. dex resolves, installs, and syncs skills, commands, rules, and MCP servers across Claude Code, Cursor, and GitHub Copilot from a single HCL definition.

Get Started GitHub

How it works

dex sits between your package sources and your AI platforms. It resolves dependencies, downloads packages, and installs resources into each platform's expected directory structure. The entire workflow is driven by two files: dex.hcl (what to install) and dex.lock (exact resolved versions).

Registries git, https, s3, az://
→
dex sync resolve · fetch · install
→
Claude Code .claude/ · CLAUDE.md
Cursor .cursor/ · AGENTS.md
GitHub Copilot .github/ · .vscode/
dex.hcl 
project {
  default_platform = "claude-code"
}

plugin "python-tools" {
  source  = "git+https://github.com/example/python-tools.git"
  version = "^1.0.0"
}

plugin "team-standards" {
  source  = "az://myregistry/packages/"
  version = "~2.3.0"
}

What dex gives you

Dependency resolution

Semver-aware resolver handles ^1.0, ~2.3, and exact version constraints. Transitive dependencies are resolved automatically. A dex.lock file pins exact versions and checksums for deterministic installs across your team.

Multi-registry support

Install packages from Git repositories, HTTPS endpoints, S3 buckets, Azure Blob Storage, or local filesystem paths. Publish your own packages to any of these backends with dex publish.

HCL configuration

First-class HCL with file(), templatefile(), and env() functions. Variables, conditional resources, and multi-file composition. Packages are defined in package.hcl, projects in dex.hcl.

Platform-aware mapping

17 resource types across three platforms. dex translates a single skill block into .claude/skills/, .cursor/rules/, or .github/prompts/ depending on your target. MCP servers, settings, and rules all map correctly.

Safe, non-destructive installs

dex tracks which files it manages via a manifest. It refuses to overwrite files it didn't create. dex sync --dry-run previews all changes. dex clean removes only managed files. Use --force to override when needed.

Personal plugins

Add resources locally via ~/.dex/local.hcl without touching the project config. Personal plugins install alongside team packages but never land in version control. Use --git-exclude to hide managed files from git status.

Install

macOS & Linux

$ curl -fsSL https://dexartifactsproduction.z13.web.core.windows.net/install.sh | bash

Windows (PowerShell)

PS> irm https://dexartifactsproduction.z13.web.core.windows.net/install.ps1 | iex

Installs to ~/.bin/dex. Add ~/.bin to your PATH if needed.

Quick Start

1
Initialize

Create a dex.hcl in your project

# Initialize for Claude Code (default)
dex init

# Or target a specific platform
dex init --platform cursor
2
Configure

Add plugins from any source

dex.hcl 
plugin "python-tools" {
  source  = "git+https://github.com/example/python-tools.git"
  version = "^1.0.0"
  config = {
    python_version = "3.12"
  }
}
3
Sync

Resolve and install everything

# Install all plugins
dex sync

# Preview changes first
dex sync --dry-run
4
Done

Skills are live in your AI assistant

Open Claude Code, Cursor, or Copilot. Installed skills, commands, rules, and MCP servers are ready to use immediately.
