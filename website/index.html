dex.

Write once. Deploy to any AI platform.

dex is a universal package manager for AI coding agents. Define skills, commands, rules, and MCP servers once in HCL and sync them to Claude Code, Cursor, or GitHub Copilot with a single command.

Get Started → GitHub

How it works

AI platforms each have their own conventions for skills, rules, commands, and MCP servers, with no standard way to share them across tools. dex bridges that gap. Write your AI packages once in package.hcl and dex handles installation into whichever platform your project targets.

Your package

  • Define skills and commands in HCL
  • Add MCP server configurations
  • Set permissions and variables
→
package.hcl one config · all platforms
→

Any AI platform

  • Claude Code (.claude/)
  • Cursor (.cursor/)
  • GitHub Copilot (.github/)

Features

📦

Universal packages

Write a package.hcl once and target multiple AI platforms. Same skill content, deployed everywhere your team works.

🗂️

Registry support

Publish packages to local directories, Git repositories, HTTPS servers, S3 buckets, or Azure Blob Storage. Install with a version constraint.

🔒

Lock file

Deterministic installations via dex.lock. Exact versions and checksums are recorded so your team always gets the same packages.

✏️

Personal tweaking

Add resources locally without touching the project config. Define personal plugins in ~/.dex/local.hcl. Only you see them and nothing lands in git.

👁️

Git exclude

Use dex sync --git-exclude to hide managed files from git status via .git/info/exclude. Share packages without polluting the tracked file list.

⚙️

HCL config

First-class HCL with file(), templatefile(), and env() functions. Templates, variables, and per-platform resources, all in one language.

🛡️

Conflict protection

dex refuses to overwrite files it didn't install. Untracked files are safe until you explicitly pass --force.

🌐

Multi-platform mapping

17 resource types spanning Claude Code, Cursor, and GitHub Copilot. Skills, commands, rules, prompts, agents, subagents, settings, and MCP servers.

Quick Start

1
Step 1

Initialize your project

Run dex init in your project directory. Creates a dex.hcl configured for your chosen AI platform.

# Initialize for Claude Code (default)
dex init

# Initialize for a specific platform
dex init --agent cursor
dex init --agent github-copilot
2
Step 2

Add plugins to dex.hcl

Edit dex.hcl to add plugins from a registry, a git repository, or a local path.

dex.hcl 
project {
  agentic_platform = "claude-code"
}

plugin "python-tools" {
  source  = "git+https://github.com/example/python-tools.git"
  version = "^1.0.0"
  config = {
    python_version = "3.12"
  }
}
3
Step 3

Sync your plugins

Run dex sync to install all plugins. dex resolves versions, downloads packages, and installs resources into your AI platform's config directories.

# Install all plugins from dex.hcl
dex sync

# Preview what would change
dex sync --dry-run

# Install a specific plugin on the fly
dex sync my-plugin@^1.0.0
4
Step 4

Use in your AI assistant

Skills, commands, and rules are now live in your AI coding assistant. Open Claude Code, Cursor, or GitHub Copilot and start using your installed packages immediately.
